<h3>1. 객체지향 프로그래밍의 의미</h3>

<br>

객체 지향 프로그래밍(영어: **o**bject-**o**riented **p**rogramming, OOP)은 객체 개념을 기반으로 하는 컴퓨터 프로그래밍의 패러다임이다, 라고 위키피디아에서 설명한다.
<br>

근데 솔직히 이게 무슨 말인지 모르겠어서,다른 쉬운 말로 바꿔보았다. <br>
쉽게 말해 코드 한 줄 한 줄 내려가며 읽는 것이 아닌, 플레이어, 맵, 적 등으로 카테고리로 나누어 한데 묶어 프로그래밍 한다는 말이다. 

<br>

<h3>2. 구성요소</h3>
클래스, 객체, 메세지로 이루어져있다. 
아오오니 저택을 예시를 들자면

<br>
<br>
플레이어, 몬스터 << 이런 게 클래스 // 카테고리같은 느낌 <br>
히로시, 아오오니 << 객체 // 클래스에 속해있는 데이터 <br>
(아오오니) 플레이어 발견시 그를 쫓아간다 << 이런 게 메세지다. //객체에 주어진 코드 <br>
