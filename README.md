<h3>1. 객체지향 프로그래밍의 의미</h3>

객체 지향 프로그래밍(영어: **o**bject-**o**riented **p**rogramming, OOP)은 객체 개념을 기반으로 하는 컴퓨터 프로그래밍의 패러다임이다, 라고 위키피디아에서 설명한다.<br>
근데 솔직히 이게 무슨 말인지 모르겠어서,다른 쉬운 말로 바꿔보았다. <br>
쉽게 말해 코드 한 줄 한 줄 내려가며 읽는 것이 아닌, 플레이어, 맵, 적 등으로 카테고리로 나누어 한데 묶어 프로그래밍 한다는 말이다. <br>
객체지향 프로그래밍을 사용하는 언어로는 c++ c# 파이썬 등으로 우리가 흔히 사용하는 대부분의 언어가 있다.

<br>

<h3>2. 구성요소</h3>
클래스, 객체, 메세지로 이루어져있다. <br>
아오오니 저택을 예시를 들자면

<br>
<br>
플레이어, 몬스터 << 이런 게 클래스 // 카테고리같은 느낌 <br>
히로시, 아오오니 << 객체 // 클래스에 속해있는 데이터 <br>
(아오오니) 플레이어 발견시 그를 쫓아간다 << 이런 게 메세지, 메소드다. //객체에 주어진 코드 <br>

<br>

<h3>3. 특징</h3>
<h4>3-1. 캡슐화</h4>
하나의 객체 안에 데이터와 기능을 묶어 외부에서 함부로 접근하지 못하게 막는 것. 
<br>

<h4>3-2. 상속</h4>
공통 기능을 만들어 여러 객체가 공유하며 사용하는 것. 예를 들어 이동기능을 만들어 그 하위에 플레이어블 캐릭터와 아오오니를 상속시킨다.

<br>
<h4>3-3. 추상화</h4>
시스템을 단순히 시켜 중요기능만 노출하는 것. 플레이어의 이속을 높이려면 코드를 바꾸고 뭐 해야할 것이 많은데 그 많은 걸 단지 대쉬 키 하나 누르는 것에 함축해 담았다. 
<br>

<h4>3-4. 다형성</h4>
같은 명령을 내려도 다르게 행동하는 것으로, 상호작용 키 하나를 누르는 것으로 문을 열거나 접시 조각을 주우는 등 다양한 행동을 하는 것을 말한다.

<br>
<br>
<h3>4. 장단점</h3>
<h4>장점</h4>
- 상속화라는 특징 덕에 코드를 가져다 쓰기 좋다. <br>
- 클래스 단위로 구분하니 수정하기에 훨씬 편하다. <br>
- 독립적인 객체 단위로 구분하기에 대규모 프로젝트에 적합하다.
<br> <br>
<h4>단점</h4>
- 객체가 많으면 많을 수록 용량이 크게 증가한다.<br>
- 설계시에 긴 시간과 노력이 들며 실행 속도가 느리다. <br>
